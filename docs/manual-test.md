# 動作確認手順

CI(`android-build` / `e2e-test`)で自動検証できない部分を、人がエミュレータまたは実機で確認するための項目。

**特に重要なのは次の2つ。** どちらもE2Eでは検証できない。

- パスワードの生体認証(`BiometricPrompt` はシステムUIのため操作できない)
- 入力フォームとキーボードの重なり(画面上の位置関係は自動判定できない)

## 実行前の準備

### 実機で確認する場合

[動作確認用APK](https://github.com/doryu0-04092/setup-assistant/releases/tag/dev) をスマホのブラウザで開いてインストールする。master の最新版が自動でビルドされている。

### Android Studio で確認する場合

1. 最新を取得する(`git pull`)
2. **File → Sync Project with Gradle Files** を実行する
   - 依存を追加したときは必須。Syncせずに実行すると古い状態のまま動く
3. Device Manager で仮想デバイスを起動する(API 34以降を推奨)
4. ▶ Run 'app' で実行する

## 0. 起動時の本人確認

端末に画面ロック(PIN/指紋)を設定した状態で確認する。

- [ ] アプリを開くと認証が求められ、**通すまで中身が見えない**
- [ ] 認証をキャンセルすると「ロックを解除してください」の画面が出る
- [ ] 「認証する」を押すと再度求められ、通すと中身が表示される

画面ロックを解除した状態(Settings → Security → Screen lock → None)でも確認する。

- [ ] 認証を求められずそのまま開ける(確認する手段がないため素通しにしている)

## 1. セットアップタブ

### 一覧

- [ ] フェーズが 0〜8 の9枚のカードで表示される
- [ ] フェーズ7に「立ち上げ完了」、フェーズ8に「自己対処可能」のラベルが付く
- [ ] 分岐のあるフェーズ(2〜6)は、未選択のうち「現状の確認が必要です」と表示される
- [ ] 分岐のないフェーズ(0・1・7・8)は進捗バーと「0 / n 完了」が表示される

### 詳細

- [ ] カードをタップすると詳細に遷移し、上部のタイトルがフェーズ名に変わる
- [ ] 左上の戻るボタンで一覧に戻れる。端末の戻る操作でも戻れる
- [ ] **「入っていない」「入っている」を切り替えると、表示される手順が変わる**
  - Git フェーズが分かりやすい。「入っていない」ならインストール手順、「入っている」なら現在の設定を確認する手順が出る
- [ ] ターミナル操作のステップは黒地・等幅のブロックで表示され、「ターミナルで操作」と示される
- [ ] 画面操作のステップは「画面上で操作」と示される
- [ ] コマンドのコピーボタンを押すと、クリップボードに入る(メモアプリ等に貼って確認)
- [ ] 「できたかどうかの確認」が各ステップに表示される
- [ ] 「公式サイトを開く」でブラウザが開く

### フェーズの一括完了

- [ ] 詳細の上部に「このフェーズを完了にする (n / n)」ボタンが表示される
- [ ] 押すと表示中の全ステップにチェックが入り、ボタンが「完了済み — 取り消す」に変わる
- [ ] 一覧に戻ると、そのフェーズの進捗が n / n になっている
- [ ] 「取り消す」を押すと全チェックが外れる
- [ ] **分岐が未選択のフェーズ(2〜6を初期状態で開く)では、ボタンが表示されない**
- [ ] 一括完了した後で分岐を切り替えると、切り替え先のステップは未完了のまま表示される

### メモと手順の上書き

- [ ] ステップ右上の編集ボタンで全画面のフォームが開く
- [ ] メモを書いて保存すると、カード上にメモが表示される
- [ ] 説明やコマンドを書き換えて保存すると「編集済み」バッジが出る
- [ ] 「書き換えを元に戻す」で標準の手順に戻り、バッジが消える。**メモは残る**
- [ ] 元の文面と同じ内容で保存した場合はバッジが出ない

### セットアップの初期化

- [ ] 上部バーの初期化ボタンで確認画面が出る
- [ ] 「初期化する」で、チェック・分岐の選択・**メモ・書き換えた手順が全て消える**
- [ ] アカウントの登録内容は消えない
- [ ] 「キャンセル」では何も消えない

## 2. アカウントタブ

- [ ] 右下の + ボタンで全画面のフォームが開く
- [ ] ラベルが空だと保存ボタンが押せない
- [ ] 二要素認証の方式をチップで選べる
- [ ] 保存するとカードとして一覧に表示される
- [ ] **一番下までスクロールしても、最後のカードが右下の + ボタンに隠れない**(パスワード欄まで読める)
- [ ] 「手順で使う」チップを押すと、そのプロファイルが使用中になる(背景色が変わる)
- [ ] アカウントIDやメールアドレスのコピーボタンが動く
- [ ] 削除するとカードが消える

### 入力フォームとキーボードの重なり(最重要)

**画面上の位置関係は自動判定できないため、目で見るしかない。** 一度この不具合を出しているので、フォームを触ったときは必ず確認する。

メモ編集とアカウント編集の両方で見る。

- [ ] フォームが画面の**下端まで**表示されている(途中で切れていない)
- [ ] 一番下の入力欄をタップしても、その欄がキーボードに隠れず見える
- [ ] キーボードが出た状態で、**画面右上の「保存」が押せる**
- [ ] キーボードが出た状態でも、上下どちらの欄までもスクロールして到達できる
- [ ] 左上の × で閉じられる

### コマンドへの差し込み

- [ ] メールアドレスを登録した状態で、セットアップタブ → フェーズ2「Git」→「入っている」または「入っていない」を選び、`git config --global user.email` のステップを見る
  - **登録したメールアドレスがコマンドに入っている**こと
- [ ] プロファイルを登録していない、またはメールアドレスが空の場合、`<登録したメールアドレス>` と表示されること

## 3. パスワードと生体認証

ここが最も確認が必要な箇所。3パターン試す。

### 準備: エミュレータに画面ロックを設定する

エミュレータの Settings → Security → Screen lock → PIN で、任意のPIN(例: 1234)を設定する。

指紋も試す場合は、続けて Settings → Security → Fingerprint で登録する。登録中に指紋を求められたら、エミュレータ右側のツールバー **…(Extended controls) → Fingerprint → Touch Sensor** を押す。

### パターンA: 画面ロックあり・PINで認証

- [ ] アカウント編集でパスワードを入力して保存 → **認証ダイアログが出る**
- [ ] PINを入力すると保存される
- [ ] カードの「認証して表示」を押す → 認証ダイアログが出る
- [ ] PINを入力するとパスワードが表示される
- [ ] 「隠す」で再び伏せ字に戻る
- [ ] コピーボタンでクリップボードに入る

### パターンB: 指紋で認証

- [ ] 認証ダイアログが出た状態で、Extended controls → Fingerprint → **Touch Sensor** を押すと認証が通る

### パターンC: 画面ロックなし

画面ロックを解除した状態(Settings → Security → Screen lock → None)で確認する。

- [ ] パスワードを入力して保存しようとすると、**「端末に画面ロック(PINや指紋)を設定してください」と案内される**
- [ ] アプリが落ちない
- [ ] パスワード以外の項目(メールアドレスなど)は、画面ロックなしでも保存・表示できる

## 4. 安全な進め方タブ

- [ ] 情報の種別ごとに「保存する / しない」が表示される
- [ ] 「なぜトークンを端末に置かないのか」の説明が表示される
- [ ] クリーンアップ項目が一覧表示される

## 5. 永続化

**アプリを完全に終了してから**再度起動して確認する(タスク一覧からスワイプで終了)。起動時の本人確認を通してから中身を見る。

- [ ] ステップのチェック状態が残っている
- [ ] 「入っていない / 入っている」の選択が残っている
- [ ] メモと手順の上書きが残っている
- [ ] アカウントの登録内容が残っている
- [ ] パスワードが残っている(認証すれば表示できる)

## 6. タブ間の移動

- [ ] タブを切り替えても、セットアップタブのスクロール位置や開いていた画面が保たれる
- [ ] フェーズ詳細を開いた状態で別タブへ移動し、セットアップタブに戻ると詳細画面のままになっている

## E2Eが自動で見ている範囲

次はCI(`e2e-test`)がエミュレータ上で毎回検証しているため、手動では省いてよい。壊れればPRの時点で分かる。

一覧の表示とスクロール、詳細への遷移と戻る、現状確認による手順の切り替え、フェーズの一括完了と取り消し、初期化(メモが消えることを含む)、タブ移動、アカウント登録とコマンドへの差し込み、キーボードが出た状態での保存ボタンへの到達。

## 確認できないこと(既知の制約)

- ログやクラッシュレポートにメモ・パスワードが出ていないことは、目視では確認できない。コードレビューで担保する
- 暗号化されて保存されていること自体は、`adb` で `/data/data` を覗ける環境(root化エミュレータ)でないと確認できない。実施する場合は EncryptedSharedPreferences のファイルが平文で読めないことを見る
- **入力欄がキーボードに物理的に覆われていないか**は、要素の位置関係を判定できないため自動化できない。E2Eは「保存に到達できる」ところまでしか保証しない
