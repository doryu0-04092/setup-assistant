# setup-assistant

現場のPCで開発環境(Cursor / Claude Code / GitHub)を立ち上げるための、手順ナビゲーターアプリ(Android)。

Android アプリ開発を一通り経験することを目的に作った。**動作する状態まで実装し、配布の仕組みも用意した上で、正式リリースは行っていない**(理由は[このリポジトリの位置づけ](#このリポジトリの位置づけ))。

## 何をするアプリか

常駐先や配属先のPCで、**指定されたアカウント**を使って開発環境をゼロから立ち上げる作業を、スマホの手順書としてナビゲートする。

- **秘密情報は端末に保存しない** — APIキーやトークンは現地で発行し、その場でPC側に設定する
- **PCがまっさらとは限らない前提** — 各フェーズの冒頭で「入っているか」「どのアカウントか」を確認し、未導入 / 導入済み(切り替えのみ)で手順が分岐する
- **ターミナル操作とUI操作を区別して表示** — どこで何をするのかが一目で分かる
- 分からない人がこれだけ見て設定できる水準を品質基準にしている

到達目標は二段階。まず **clone → 編集 → commit → push が通る**(実務環境の立ち上げ完了)、次に **AIとGitHubで自分で調べ・解決していける**(自己対処可能)状態にする。

## 試す

**[動作確認用のAPKをダウンロード](https://github.com/doryu0-04092/setup-assistant/releases/tag/dev)** — master の最新版から自動でビルドしている。スマホのブラウザでこのページを開けば、そのままインストールできる。

### インストール手順

1. 上のリンクをAndroid端末で開き、`setup-assistant-dev.apk` をダウンロードする
2. ダウンロードしたファイルを開く
3. 「提供元不明のアプリ」に関する確認が出たら、案内に従って設定画面で許可する
   - Google Play 以外から入れるアプリは既定では止められるため。ブラウザやファイルアプリごとに許可する形になっている
4. 戻ってインストールし、起動する

動作要件: **Android 8.0 以降**

> デバッグ用の署名で配布している。正式リリース版は公開していない。

## 画面

| タブ | 内容 |
|---|---|
| セットアップ | 9フェーズの手順。各フェーズ冒頭の現状確認で、未導入 / 導入済みに手順が分岐する |
| アカウント | 使うアカウントの識別子を控える。登録した値は手順のコマンドに差し込まれる |
| 安全な進め方 | 何を保存し何を保存しないか、現場PCを離れるときに消すもの |

## 開発

| CI | 内容 |
|---|---|
| `android-build` | push ごとに `gradle assembleDebug` を実行する |
| `e2e-test` | エミュレータ上でCompose UI Testを実行し、実際の画面操作を検証する |
| `link-check` | 手順内の公式URLを週次で確認する(URLは `SetupContent.kt` から自動抽出) |
| `release` | `v*` のタグを打つと署名付きAPKをビルドして Releases に添付する |

実装は issue → ブランチ → PR で進める。

パスワードまわりは `BiometricPrompt` がシステムUIのためE2Eで検証できず、[動作確認手順](docs/manual-test.md) の手動確認で担保している。

## このリポジトリの位置づけ

Android 開発の学習として作ったもので、**当初予定していたコンテストには別のアイデアで臨むことにしたため、正式リリースはしない**。

ただし途中で止めたわけではなく、次の状態まで作ってある。

| | 状態 |
|---|---|
| 機能 | 一通り動く。9フェーズの手順、アカウント情報、生体認証、進捗とメモの保存 |
| 検証 | ビルドとE2E(11件)がCIで自動実行される。手動確認の項目も文書化済み |
| 配布 | 署名付きAPKをタグで公開する仕組みまで用意。**鍵を登録すればリリースできる状態**で止めてある |

動作を見たい場合は上の[動作確認用APK](#試す)からインストールできる。

途中で設計の前提が誤っていると分かり、方針を転換している。その経緯は [コンセプト](docs/concept.md) の「方針転換の記録」に残した。
